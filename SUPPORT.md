# Support

## Channels

| Channel | Use |
| --- | --- |
| GitHub issues | Reproducible bugs, documentation corrections, project tracking |
| GitHub profile | Maintainer contact and project routing |
| LinkedIn | Professional inquiries and commercial discussion |

## Support Scope

| In Scope | Out Of Scope |
| --- | --- |
| Reproducible theme defects | General WordPress training |
| Packaging/documentation issues | Third-party plugin support not shipped by the package |
| Release and repository issues | Custom feature development without agreed scope |
| Compatibility-safe maintenance | Emergency support guarantees |

## Bug Report Checklist

1. Theme version and commit or release tag.
2. WordPress and PHP version.
3. Active plugins relevant to the issue.
4. Reproduction steps.
5. Expected result.
6. Actual result.
7. Screenshots or logs if relevant.

## Issue Expectations

- Keep one issue per defect.
- Include whether the issue occurs with default WordPress settings.
- Call out whether Elementor, Redux, or ACF is involved.
